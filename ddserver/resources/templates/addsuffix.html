Add a new zone

Using the form on the right, you can add a new DNS zone, within which users are allowed to register hostnames. If you have multiple zones configured, a user may choose within which zone a hostname should be added. By adding the zone example.org, users may create hostnames like myhost.example.org.

Prerequisites

To be able to resolve hostnames in the new zone, the PowerDNS server connected with ddserver must be configured to be authoritative for the zone you add here. Please refer to the ddserver documentation for more information on zone delegation.

Important notes

Be aware that users are allowed to add any subdomain (i.e. mail) to the zone. Please refer to the ddserver documentation to get information on how to restrict specific hostnames from being registered.

Further, be aware that no duplicate FQDNs are allowed, which means that none of the hostnames in this zone configured in ddserver may appear in any other of your PowerDNS backends.
